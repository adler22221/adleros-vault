# Using an old MacBook Air for a home server – Decoding
I just assembled this temporary home server “rack” with my 2018 MacBook Air on top.

![MacBook Air server on a self](https://decoding.io/wp-content/uploads/2021/12/IMG_0305-scaled.jpeg "IMG_0305.jpeg")

It does the following things:

-   General file sharing and project archival
-   Backups my iCloud Drive and Photos to a Time Capsule
-   Runs [Time Machine server](https://support.apple.com/en-au/guide/mac-help/mchl31533145/mac) for my M1 MacBook Pro
-   Tests Rails code against Intel via a local [GitHub runner](https://docs.github.com/en/actions/using-github-hosted-runners/about-github-hosted-runners) instance
-   Processes files via [DEVONthink smart rules](https://www.devontechnologies.com/blog/20201110-organize-with-smart-rules)
-   Schedules some of my RSS feeds which are published via [Feeder](https://reinventedsoftware.com/feeder/)
-   Archives my websites using rsync
-   Filters my email via [Mail rules](https://support.apple.com/guide/mail/use-rules-to-manage-emails-you-receive-mlhlp1017/mac)
-   Keeps my [Reeder](https://reederapp.com/) queue up-to-date
-   [Content caching](https://support.apple.com/guide/mac-help/what-is-content-caching-on-mac-mchl9388ba1b/mac)
-   [Local device backups](https://setapp.com/how-to/backup-iphone-to-external-drive) for my iPad Pro and iPhone
-   Media server based on the [Apple TV app and Home Sharing](https://support.apple.com/en-us/HT202190)
    -   I don’t care about Plex.
-   Torrent server to acquire the latest Linux ISOs
-   Connects my [DYMO labeler](https://www.dymo.com/label-makers-printers/labelmanager-label-makers/dymo-labelmanager-plug-n-play-label-maker/SAP_1768960.html), because it caused kernel panics on my M1 MacBook Pro

I plan to get a proper Mac mini after we moved into our new house, but for now, this is more than sufficient.
